**Runtime** (environment) is extra code added by the compiler (without your knowledge) to make the language behave in the way that you expect.

Extra code and extra library to make the code behave the way you expect it to behave.

Key aspects of a runtime system include:
• **Stack management** – Ensures local variables exist only within their function.
• **Memory handling** – Manages allocation and deallocation (e.g., garbage collection).
• **Error handling** – Detects and manages runtime errors like division by zero.
• **OS interaction** – Provides built-in support for file handling, networking, etc.
