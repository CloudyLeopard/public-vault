### MPI_MAXLOC and MINLOC

`MPI_MAXLOC`: returns the maximum and the rank of the process that has the maximum
`MPI_MINLOC`: returns the minimum and the the rank of the process that has the minimum
> cuz when you call `MPI_Reduce` every rank passes in a variable. it basically looks at which rank's variable has the highest value

Recall in `MPI_Reduce` the output is one number... how do you get two numbers?

> [!note] `MPI_MAXLOC` and `MPI_MINLOC`
> In order to use `MPI_MINLOC` and `MPI_MAXLOC` one must provide a datatype argument that represents a pair
> MPI provides the following pair datatypes:
> ![[mpi pair datatypes.png|400]]
> Note: the rank is always an int, which is why the second number in the pair is an int.
> This is something you supply into the parameter `datatype`

**Example**:


## Collective vs Point-to-Point Communications
**Collective**
All the processes in the communicator must call the same collective function

The arguments passed by each process to an MPI collective communication must be "compatible"
- For example, if one process passes in 0 as the `dest_process` and other apsses in 1, then the outcome of a call to `MPI_Reduce` is errorneous
- The only exception is the second one (`output_data_p`)

The `output_data_p` argument is only used on `dest_process`
> this data structure may also exist in other processes but they will not be touched

However, all of the processes still need to pass in an actual argument corresponding to `output_data_p`, even if it is NULL

**All collective function calls are blocking** (*i think, double check slides*)

### Keep in mind
- In distributed memory systems, communications is more expensive than computation
- Distributing a fixed amount of data among several messages is more expensive than sending a single big message
> Sending one message of 100 bytes is much cheaper (take much less time) than sending 10 messages of 10 bytes each.
> Reason 1: all of these communciatinos end up being OS calls, and OS system calls end up being expensive. Sending one message - only sending OS calls once. 
> Reason 2: when the MPI system is sending message from one point to the other, its not just adding one message but also sending header and footer to be routed correctly etc. this is all overhead, and this over head gets multipled if we send multiple messages
