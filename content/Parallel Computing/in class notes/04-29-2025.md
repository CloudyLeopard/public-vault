### Computation vs Memory Access

Compute to global memory access (CGMA) ratio
**Definition**: The number of FP calculations performed for each access to the global memory within a regioin in a CUDA program.

> each computation, how many global memory access we do. if we find 10 computation, and we went to global emory twice, CGMA = 10/2 = 5. This means for each tie you go to memory, you do 5 computations on the GPU.

Given this definition, is CGMA the higher the better or the lower the better
A: higher! doing computation is good, while going to global memory is bad. Always want more computation and less global memory access.

## Memory As Limiting Factor to Parallelism
Limited shared memory limits the number of threads that can execute simultaneously in SM for a given application
> remember one SM can host one more more blocks; but a block cannot be hosted in an SM if not all the resources are loaded. If one block uses al the shared memory in an SM, no other blocks can be assigned.
> since all the blocks are the same - they all require the same shared memory
- the more memory locations each thraed quires, the fewer the number of of threads per SM
- same applies to registers
> if one block uses most of the registers, another block cannot share the same SM as that block because it will nto have the registers it needs. registers and shared memories are both resources. 

A block cannot be assigned on an SM unless it gets all the resource it needs

### Examples:

EXAMPLE: Registers
- G80 has 8K registers per SM -> 128K registers for entire processors
> G80 has 16 SMs; 8K * 16 = 128K
- G80 can accomodate up to 768 threads per SM
- To fill this capacity each thread can use only 8K/768 = 10 registers
- If feach thread uses 11 registers -> threads per SM are reduced -> **per block granularity**
- e.g. if block contains 256 threads the number of threads will be reduced by 256 -> lowering the number of threads/SM from 768 to 512 (i.e. 1/3 reduction of threads!)

EXAMPLE: Shared Memory
- G80 has 16Kb of shared memory per SM
- SM accomodates up to 8 blocks
- To reach this maximum each block must not exceed 16Kb/8 = 2KB of memory
- e.g. if each block uses 5KB -> no more than 3 blocks can be assigned to each SM
